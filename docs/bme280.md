# BME280 Temperature / Humidity / Pressure

Reads temperature, humidity and barometric pressure from a Bosch BME280 over I2C. Shares the library I2C bus.

**Platforms:** `esp32` `esp32s2` `esp32s3` `esp32c3` `esp32c6`

**Requires:** i2c

## Entities

| Domain | Name |
|---|---|
| `sensor` | Temperature |
| `sensor` | Humidity |
| `sensor` | Pressure |

## Usage

Local include (repo checked out beside your config):

```yaml
packages:
  bme280: !include
    file: esphome_sensor_templates/templates/environment/bme280.yaml
    vars: { st_update_interval: 60s }
```

Remote include (pulled straight from GitHub):

```yaml
packages:
  bme280: github://OWNER/esphome_sensor_templates/templates/environment/bme280.yaml@main
```

> Replace `OWNER` with the GitHub owner/repo that hosts this library.

## Variables

| Variable | Default | Description |
|---|---|---|
| st_update_interval | `60s` | How often to sample the sensor |
| st_name_prefix | `""` | Prefix prepended to every entity name |
| st_disabled_by_default | `false` | Ship the entities disabled in HA |
| st_internal | `false` | Hide the entities from HA/web entirely |
| st_state_class | `measurement` | state_class for all readings; set "" to opt out of HA long-term statistics |
| st_bme280_i2c_id | `st_i2c_bus` | id of the I2C bus to use (from core/i2c.yaml) |
| st_bme280_address | `0x76` | I2C address (0x76 SDO->GND, 0x77 SDO->VCC) One of: `0x76`, `0x77`. |

## Notes

- Requires the shared I2C bus - include core/i2c.yaml. Sub-sensors keep the component default 16x oversampling; override via !extend for lower power.
- A BMP280 (no humidity) is a different chip - use the bmp280_i2c platform instead. A "BME280" that reports 0% humidity is usually a mislabelled BMP280.
