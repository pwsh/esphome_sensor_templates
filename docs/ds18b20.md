# DS18B20 Temperature

Reads a Dallas DS18B20 temperature sensor over a dedicated 1-Wire bus. Owns its one_wire: bus (the only template that uses it).

**Platforms:** `esp32` `esp32s2` `esp32s3` `esp32c3` `esp32c6`

**Requires:** none

## Entities

| Domain | Name |
|---|---|
| `sensor` | Temperature |

## Usage

Local include (repo checked out beside your config):

```yaml
packages:
  ds18b20: !include
    file: esphome_sensor_templates/templates/environment/ds18b20.yaml
    vars: { st_update_interval: 60s, st_ds18b20_pin: <value> }
```

Remote include (pulled straight from GitHub):

```yaml
packages:
  ds18b20: github://OWNER/esphome_sensor_templates/templates/environment/ds18b20.yaml@main
```

> Replace `OWNER` with the GitHub owner/repo that hosts this library.

## Variables

| Variable | Default | Description |
|---|---|---|
| st_update_interval | `60s` | How often to read the sensor |
| st_name_prefix | `""` | Prefix prepended to the entity name |
| st_disabled_by_default | `false` | Ship the entity disabled in HA |
| st_internal | `false` | Hide the entity from HA/web entirely |
| st_state_class | `measurement` | state_class for the reading; set "" to opt out of HA long-term statistics |
| **st_ds18b20_pin** | **(required)** | GPIO connected to the 1-Wire DATA line |
| st_ds18b20_resolution | `12` | ADC resolution in bits (higher = slower conversion) One of: `9`, `10`, `11`, `12`. |

## Notes

- A 4.7kΩ pull-up between DATA and 3.3V is required.
- This preset assumes ONE sensor on the bus (no address needed - it auto-selects the single device). For multiple DS18B20s on one bus, give each its own address via !extend st_ds18b20_temperature (address: 0x...) and add more dallas_temp sensors on the same st_ds18b20_one_wire bus. Run once with logs to read the discovered addresses.
