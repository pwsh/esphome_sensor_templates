# LD2450 mmWave Tracking

HiLink LD2450 24GHz mmWave radar with multi-target tracking over UART. Exposes occupancy plus the active target count. Owns its own uart: bus.

**Platforms:** `esp32` `esp32s2` `esp32s3` `esp32c3` `esp32c6`

**Requires:** none

## Entities

| Domain | Name |
|---|---|
| `binary_sensor` | Presence |
| `sensor` | Target Count |

## Usage

Local include (repo checked out beside your config):

```yaml
packages:
  ld2450: !include
    file: esphome_sensor_templates/templates/presence/ld2450.yaml
    vars: { st_name_prefix: "", st_ld2450_uart_tx_pin: <value>, st_ld2450_uart_rx_pin: <value> }
```

Remote include (pulled straight from GitHub):

```yaml
packages:
  ld2450: github://OWNER/esphome_sensor_templates/templates/presence/ld2450.yaml@main
```

> Replace `OWNER` with the GitHub owner/repo that hosts this library.

## Variables

| Variable | Default | Description |
|---|---|---|
| st_name_prefix | `""` | Prefix prepended to every entity name |
| st_disabled_by_default | `false` | Ship the entities disabled in HA |
| st_internal | `false` | Hide the entities from HA/web entirely |
| st_state_class | `measurement` | state_class for the target-count sensor; set "" to opt out of HA long-term statistics |
| **st_ld2450_uart_tx_pin** | **(required)** | GPIO wired to the sensor RX pin (ESP TX) |
| **st_ld2450_uart_rx_pin** | **(required)** | GPIO wired to the sensor TX pin (ESP RX) |

## Notes

- The LD2450 uses a fixed 256000 baud UART (parity NONE, 1 stop bit); firmware V2.02.23090617 or later is required. Use hardware UART pins. ESP32 has 3 hardware UARTs, C3/C6 have 2 - one is usually taken by the logger (UART0).
- The LD2450 tracks up to 3 targets and supports zones. This preset keeps it light (presence + count). Add per-target position via the ld2450 sensor platform (target_1 x/y/speed/distance/angle, target_2..3), or zone counts, with ld2450_id: st_ld2450 - or !extend st_ld2450 for install/multi-target/tracking-mode config.
